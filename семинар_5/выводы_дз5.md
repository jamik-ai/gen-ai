# Отчёт ДЗ5 — Макро-агент: тестирование и диагностика

## 1. Новый инструмент `compare_periods`

**Сигнатура:**
```python
def compare_periods(
    metric: str,     # "key_rate" | "fx_USD" | "fx_EUR" | "fx_CNY" | "cpi" | "unemployment"
    period_a: str,   # "YYYY-MM" или "YYYY-MM-DD"
    period_b: str,
) -> dict:
```

**Возвращает:**
```json
{
  "metric": "fx_USD",
  "a": {"date": "2022-01-01", "value": 74.68},
  "b": {"date": "2026-04-01", "value": 83.25},
  "delta": 8.57,
  "ratio": 1.1147,
  "source": "fallback_csv/cbr_live"
}
```

Внутри вызывает существующие `get_fx_rate`, `get_key_rate`, `get_inflation`,
`get_unemployment` — без дублирования логики. Поддерживает форматы дат `YYYY-MM`
и `YYYY-MM-DD`.

**Примеры запросов, где агент вызывает `compare_periods`:**

| Запрос | metric | Результат |
|--------|--------|-----------|
| «Во сколько раз вырос курс USD с января 2022 по апрель 2026?» | `fx_USD` | ratio ≈ 1.11 |
| «Как изменилась ключевая ставка с марта 2022 по март 2025?» | `key_rate` | delta = +12 пп |
| «Вырос или упал курс CNY к рублю с 2020 по сегодня?» | `fx_CNY` | ratio ≈ 1.5 |

## 3. Диагностика трёх типов ошибок

### Тип A: галлюцинация инструмента

**Описание:** агент пытается вызвать несуществующий инструмент
`get_cumulative_inflation`.

**Трасса из `trace.jsonl`:**
```json
{"run_id": "...", "step": 1, "call": "get_cumulative_inflation",
 "args": {"year_start": 2022, "year_end": 2026},
 "obs": {"error": "неизвестный инструмент: get_cumulative_inflation"}}
{"run_id": "...", "step": 2, "call": "get_inflation",
 "args": {"year": 2022, "month": 1},
 "obs": {"year": 2022, "month": 1, "cpi_yoy": 8.73, "source": "rosstat_csv"}}
```

**Причина:** системный промпт не содержал запрета выдумывать инструменты для
накопленной инфляции — модель «достраивает» удобный инструмент.

**Что исправить:** явный запрет в промпте: «ЗАПРЕЩЕНО вызывать инструменты не
из списка выше». Для накопленной инфляции — описать алгоритм через цикл `get_inflation`.

---

### Тип B: битый JSON в аргументах

**Описание:** модель возвращает аргументы с незакрытой строкой.

**Трасса из `trace.jsonl`:**
```json
{"run_id": "...", "step": 1, "call": "get_fx_rate",
 "args": {},
 "obs": {"error": "битый json аргументов: Expecting ',' delimiter: line 1 column 28 (char 27)"}}
{"run_id": "...", "step": 2, "call": "get_fx_rate",
 "args": {"currency": "USD", "on_date": "2022-01-01"},
 "obs": {"currency": "USD", "date": "2022-01-01", "rate": 74.68, "source": "fallback_csv"}}
```

**Причина:** при длинном контексте токенайзер изредка «обрезает» JSON аргументов.
Встречается редко, но не исключается даже при `temperature=0.0`.

**Что исправить:** обработка уже есть в `_exec_one` (ошибка → obs → следующий шаг).
Можно добавить валидацию аргументов на стороне клиента перед вызовом функции.

---

### Тип C: арифметика «в уме» без `calculate`

**Описание:** агент получил два числа и самостоятельно поделил их в тексте
финального ответа, не вызвав `calculate`.

**Трасса из `trace.jsonl`:**
```json
{"run_id": "...", "step": 1, "call": "get_fx_rate",
 "args": {"currency": "USD", "on_date": "2022-01-01"},
 "obs": {"rate": 74.68}}
{"run_id": "...", "step": 2, "call": "get_fx_rate",
 "args": {"currency": "USD"},
 "obs": {"rate": 83.25}}
{"run_id": "...", "step": 3,
 "final": "Курс вырос примерно в 1.11 раза (83.25 / 74.68 ≈ 1.11)."}
```

**Причина:** модель обучена выполнять простую арифметику в тексте; для
«очевидных» делений она игнорирует правило «только через calculate».

**Что исправить:** усилить формулировку в промпте: «ЛЮБОЕ деление и умножение —
только через calculate, даже если кажется тривиальным». Именно поэтому добавлен
`compare_periods` — он возвращает `ratio` сразу, снимая соблазн считать в уме.

## 4. Вывод: самый хрупкий элемент системы

**Самый хрупкий элемент — системный промпт.**

Все три типа ошибок имеют общую природу: модель «знает лучше» и обходит
ограничения из промпта. Галлюцинация инструментов, арифметика в уме, битый JSON —
это следствие того, что правила задаются текстом, а не структурными ограничениями.

Эффективные структурные меры:
1. `compare_periods` с полем `enum` для метрики — не даёт передать несуществующую метрику.
2. Валидация в `_exec_one` — ошибка возвращается агенту как `obs`, он исправляется.
3. Обязательный `calculate` через `submit_answer` (структурный финал) — агент
   не может написать число в уме без tool call.

Следующий уровень защиты — паттерн Планировщик-Исполнитель-Критик (ДЗ6):
Критик явно проверяет, не посчитал ли агент что-то без `calculate`.
